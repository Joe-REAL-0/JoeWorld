# 📝 使用说明

## 怎么用这个模板

1. **复制整个文件**，重命名为 `[角色名]设定.md`
2. **全文替换占位符**：
   - `[角色名]` → 实际角色名
   - `[物种/类型]` → 人类 / AI / 改造人 / 外星种族 等
   - `[称号/代称]` → 角色的称号、代号或别名
3. **按结构填内容**：先填"角色介绍"四段叙事，再填"详细性格侧写"七大节
4. **删掉底部使用说明**和顶部模板指引

## 哪些节必须保留

- ✅ **角色介绍**：四段叙事段落，定义角色的来历、经历、当前、与主线的关系
- ✅ **七大侧写节**（起源 / 性格核心 / 行为风格 / 内在冲突 / 关系网络 / 关键场景 / 结语）：对齐 Joe、Shadow、Monsy 三位主角的结构

## 哪些节可以省略或合并

- ⭕ **性格核心第 3 条**：只有复杂角色才需要第三维
- ⭕ **内在冲突第 2 条**：次要角色可以只保留一个冲突
- ⭕ **关系网络中的势力关系**：如果角色不与特定势力纠葛，可省略
- ⭕ **关键场景**：如果角色的关键场景在其他文档里已描述，可以用一两句话引用

## 字数建议

- **角色介绍**：800-1500 字（4 段叙事）
- **详细性格侧写**：3000-6000 字（七大节合计）
- **总字数**：4000-7500 字（与现有三位主角对齐）

## 风格对齐

参考 `Joe设定.md`、`Shadow设定.md`、`Monsy设定.md` 的写作风格：

- **第二人称叙述为主**，偶尔穿插直接引语
- **具体场景 + 抽象哲思**：每个性格特质都要有具体例子支撑
- **保留张力**：角色不应是"完美的英雄"或"纯粹的恶人"，应有矛盾与裂隙
- **结尾段独立成立**：让读者合上文档后还能回想起那段话
